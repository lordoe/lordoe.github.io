Polynomielle Regression

Wiener Wohnungspreise · Grad-Slider · TU Wien
Grundidee
Polynomielle Regression erweitert die lineare Regression durch Feature-Transformation:
x
⟶
[1, x, x², …, xⁿ]
⟶
selbe Normalengleichung: \(\mathbf{w} = (\mathbf{X}^\top \mathbf{X})^{-1}\mathbf{X}^\top \mathbf{y}\)
\[ y = w_0 + w_1 x + w_2 x^2 + \cdots + w_n x^n \]
Unteranpassung
Grad 1–2: Modell zu einfach → hoher Bias, hoher Fehler auf Trainings- und Testdaten.
Gute Anpassung
Grad 3–4: Balancierte Komplexität → geringer Bias, akzeptable Varianz.
Überanpassung
Grad ≥ 5: Kurve passt Rauschen an → sehr große Koeffizienten, hohe Varianz auf neuen Daten.

Feature-Transformation im Detail
Für jeden Datenpunkt \(x_i\) wird ein Zeilenvektor erzeugt: [1, \(x_i\), \(x_i^2\), …, \(x_i^n\)].
Diese Zeilen bilden die Design-Matrix \(\mathbf{X}\) (n Punkte × (Grad+1) Features).
Danach: selbe Normalengleichung \(\mathbf{w} = (\mathbf{X}^\top \mathbf{X})^{-1} \mathbf{X}^\top \mathbf{y}\) wie bei linearer Regression.
Folge: Polynomielle Regression ist linear in den Parametern — daher ist die Normalengleichung direkt anwendbar. Das Phänomen hoher Koeffizienten bei großem Grad führt direkt zu Ridge- und Lasso-Regularisierung. → Siehe auch: Normalengleichung (Schritt-für-Schritt Berechnung), Ridge Regression (L2), Lasso Regression (L1).
