k-Nearest Neighbours

// supervised · classification

Algorithm

k-NN classifies a new input by finding the k closest training examples in feature space and letting them vote for a class label. No model is built during training — all computation happens at classification time. This makes k-NN a lazy learner.

01
Choose k and a distance function
The two key hyperparameters. k controls how many neighbours vote; the distance function defines "closeness" in feature space.
02
Compute distances to all training points
For a query point q, compute d(q, xᵢ) for every training example xᵢ.
03
Select the k nearest neighbours
Sort by distance, take the k smallest. These neighbours form the neighbourhood of q.
04
Majority vote → predicted class
The class label that appears most often among the k neighbours is assigned to q. For weighted voting each neighbour's vote is scaled by 1/rank or 1/distance.

Distance Functions

Euclidean (L2)
\( d(\mathbf{a}, \mathbf{b}) = \sqrt{\sum_i (a_i - b_i)^2} \)
Manhattan (L1)
\( d(\mathbf{a}, \mathbf{b}) = \sum_i |a_i - b_i| \)
Minkowski (Lₚ)
\( d(\mathbf{a}, \mathbf{b}) = \left( \sum_i |a_i - b_i|^p \right)^{1/p} \)
\( L_\infty\!: \max_i |a_i - b_i| \) · \(\text{Cosine: } 1 - \frac{\mathbf{a} \cdot \mathbf{b}}{\|\mathbf{a}\| \|\mathbf{b}\|}\)

Properties & Edge Cases

Lazy Learner
No model is built during training. All computation is deferred to classification. Memory and time cost grow linearly with the dataset.
Decision Boundary
k = 1 produces a highly irregular (Voronoi) boundary that overfits noise. Larger k smooths the boundary at the cost of bias.
Noise Sensitivity
k-NN is very sensitive to local outliers. Increasing k averages over more neighbours, reducing noise effects at the boundary.
Extremes of k
k = 1 is the "nearest neighbour" algorithm. k = n (all samples) always predicts the majority class — ignoring the query completely.
