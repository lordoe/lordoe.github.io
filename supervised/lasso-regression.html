Lasso Regression

Lineare Regression mit L1-Regularisierung — kann Koeffizienten exakt auf 0 setzen und damit automatisch Feature-Selektion betreiben.

Motivation: Sparse Modelle

Bei Ridge Regression werden Koeffizienten zwar klein, aber nie exakt null. Lasso (Least Absolute Shrinkage and Selection Operator) verwendet die L1-Norm als Penalty — diese hat die besondere Eigenschaft, Koeffizienten auf exakt 0 zu stauchen. Das Modell führt damit implizit eine Feature-Selektion durch.

Problem

Viele Features sind irrelevant oder redundant — ein Modell mit allen Features ist schwer interpretierbar.

Idee

Bestrafe die Summe der Absolutwerte der Koeffizienten statt deren Quadrate.

Effekt

Irrelevante Features erhalten Koeffizient = 0. Das Modell wird sparse und interpretierbar.

Kostenfunktion
Allgemeine Idee
\[\text{Total cost} = \underbrace{\text{measure of fit}}_{\color{#4ecdc4}{\text{RSS}}} + \color{#c96a28}{\lambda} \cdot \underbrace{\text{measure of magnitude of coefficients}}_{\color{#4ecdc4}{\|\mathbf{w}\|_1}}\]
Lasso Regression — L1-Regularisierung
\[\text{Total cost} = \color{#4ecdc4}{\text{RSS}(\mathbf{w})} + \color{#c96a28}{\lambda} \cdot \|\mathbf{w}\|_1\]
mit \(\;\|\mathbf{w}\|_1 = |w_0| + |w_1| + \cdots + |w_n|\)
RSS — Residual Sum of Squares
\[\text{RSS}(\mathbf{w}) = \sum_{i=1}^{N}(y_i - \hat{y}_i)^2 = \sum_{i=1}^{N}(y_i - \mathbf{w}^\top \mathbf{x}_i)^2\]
RSS(w)

Misst wie gut das Modell zu den Daten passt. Wird durch die Kostenfunktion minimiert.

λ (Lambda)

Hyperparameter, der den Kompromiss zwischen Fit und Sparsity steuert.

\(\|\mathbf{w}\|_1\)

Summe der Absolutwerte. Die "Ecken" der L1-Kugel liegen auf den Achsen — dort sind Koeffizienten 0.

Warum werden Koeffizienten exakt 0?

Die L1-Norm ist an der Stelle w = 0 nicht differenzierbar. Geometrisch hat der zulässige Bereich (L1-Ball) spitze Ecken auf den Koordinatenachsen. Das Optimum liegt mit hoher Wahrscheinlichkeit genau in einer solchen Ecke — dort sind einzelne Koeffizienten exakt 0.

L1-Ball (Lasso)

Raute-Form in 2D. Ecken liegen auf Achsen. Das Optimum "trifft" eine Ecke → wᵢ = 0.

L2-Ball (Ridge)

Kreis-Form in 2D. Keine Ecken auf Achsen. Das Optimum liegt auf einer Kurve → wᵢ ≈ 0, aber nie exakt.

Subgradient

Statt Ableitung bei 0: Subgradient \(\partial|w| \in [-1,\,+1]\). Optimierung möglich, aber kein glatter Gradient.

Wahl von λ — Sparsity-Fit Tradeoff
Großes λ

Viele Koeffizienten = 0
Starke Regularisierung, sehr sparses Modell. Kann wichtige Features eliminieren (Underfitting).

Kleines λ

Wenig Koeffizienten = 0
Schwache Regularisierung, nähert sich normaler linearer Regression. Weniger Feature-Selektion.

\(\lambda = 0\) entspricht normaler linearer Regression ohne Regularisierung. \(\lambda \to \infty\) setzt alle Koeffizienten auf 0 (nur Intercept bleibt).

Im Gegensatz zu Ridge Regression gibt es für Lasso keine analytische Lösung, da die L1-Norm bei 0 nicht differenzierbar ist. In der Praxis werden iterative Algorithmen verwendet:

Coordinate Descent

Optimiert einen Koeffizienten nach dem anderen, hält den Rest fest. Effizient und weit verbreitet (sklearn).

Subgradient-Methoden

Verallgemeinerung des Gradientenabstiegs für nicht-differenzierbare Funktionen.

LARS-Algorithmus

Least Angle Regression — berechnet den gesamten Lasso-Pfad aller λ-Werte effizient.

Coordinate Descent Update (Soft-Thresholding)
\[w_j \leftarrow \frac{\color{#4ecdc4}{S}(\color{#4ecdc4}{\rho_j},\, \color{#c96a28}{\lambda})}{z_j}\]
mit \(S(\rho, \lambda) = \operatorname{sign}(\rho)\cdot\max(|\rho| - \lambda,\, 0)\) — der Soft-Threshold-Operator
