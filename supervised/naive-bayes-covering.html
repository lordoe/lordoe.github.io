Part 1 — Naive Bayes

A simple probabilistic classifier based on Bayes' theorem with the "naive" assumption that all features are conditionally independent given the class. Despite the strong assumption, it performs surprisingly well in practice.

Bayes' Theorem
\[ P(C \mid \mathbf{x}) = \frac{P(\mathbf{x} \mid C) \cdot P(C)}{P(\mathbf{x})} \]   posterior   = likelihood  · prior  / evidence
For classification we only need to find the class that maximises the posterior — P(x) is identical for all classes and can be dropped.
Naive Independence Assumption
\[ P(\mathbf{x} \mid C) \approx \prod_i P(x_i \mid C) \] features assumed independent given class

Each feature contributes its own conditional probability, estimated separately from training data. For discrete attributes the estimate is straightforward frequency counting.

Classification Rule
\[ \hat{C} = \arg\max_C \; P(C) \cdot \prod_i P(x_i \mid C) \] pick the class with highest score

Works well even when the independence assumption is violated. Robustness comes from the fact that we only need the relative ranking of classes, not exact probabilities.

Part 2 — Covering Algorithms (Rule Learning)

Covering algorithms learn a set of IF-THEN rules by repeatedly finding a rule that covers positive examples of a target class, then removing those examples and repeating until all are covered.

Simple Covering Algorithm
  1. Start with empty rule: If ? → class
  2. Add the test that maximises p/t (positive / total covered)
  3. Stop when p/t = 1 or no improvement possible
  4. Remove covered positive instances from dataset
  5. Repeat until all positives are covered
p/t Ratio
\[ \frac{p}{t} = \frac{\text{positive examples covered}}{\text{total examples covered}} \]

We greedily pick the test with the highest ratio. When there are ties, prefer the test that covers more positive instances (higher absolute p).

Key Insights

Rules vs. Decision Trees

  • Decision trees can have replicated subtrees in different branches
  • Rule sets avoid replication — each rule is independent
  • Rules are often more readable for multi-class problems
  • Covering concentrates on one class at a time

Naive Bayes Strengths

  • Very fast training and prediction
  • Works well with high-dimensional feature spaces
  • Handles missing data gracefully
  • Robust: needs relative ordering, not exact probabilities

Two Approaches to Rules

(1) Convert a decision tree to rules — then prune each rule independently.

(2) Generate rules directly with covering — greedily specialise until p/t = 1, then repeat for remaining instances.

When Covering Stops

  • p/t = 1.0: rule is perfectly precise
  • No test improves p/t further
  • All positive instances of the class are covered
  • Remaining instances passed to the next rule
