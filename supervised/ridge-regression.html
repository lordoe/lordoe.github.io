Ridge Regression

Lineare Regression mit L2-Regularisierung — verhindert Overfitting durch das Bestrafen großer Koeffizienten.

Motivation: Overfitting bei Polynomen

Polynome höherer Ordnung passen sich sehr flexibel an Trainingsdaten an. Dabei werden die Koeffizienten sehr groß — das Modell schwingt stark und generalisiert schlecht. Ridge Regression begegnet diesem Problem durch eine Penalty auf die Größe der Koeffizienten.

Problem

Hochgradige Polynome passen perfekt auf Trainingsdaten, versagen aber auf neuen Daten (Overfitting).

Ursache

Die Koeffizienten w werden sehr groß, um jeden Datenpunkt zu treffen — das Modell lernt das Rauschen.

Lösung

Füge einen Regularisierungsterm zur Kostenfunktion hinzu, der große Gewichte bestraft.

Kostenfunktion
Allgemeine Idee (aus den Folien)
\[\text{Total cost} = \underbrace{\text{measure of fit}}_{\color{#4ecdc4}{\text{RSS}}} + \color{#c96a28}{\lambda} \cdot \underbrace{\text{measure of magnitude of coefficients}}_{\color{#4ecdc4}{\|\mathbf{w}\|^2_2}}\]
Ridge Regression — L2-Regularisierung
\[\text{Total cost} = \color{#4ecdc4}{\text{RSS}(\mathbf{w})} + \color{#c96a28}{\lambda} \cdot \|\mathbf{w}\|_2^2\]
mit \(\;\|\mathbf{w}\|_2^2 = w_0^2 + w_1^2 + \cdots + w_n^2\)
RSS — Residual Sum of Squares
\[\text{RSS}(\mathbf{w}) = \sum_{i=1}^{N}(y_i - \hat{y}_i)^2 = \sum_{i=1}^{N}(y_i - \mathbf{w}^\top \mathbf{x}_i)^2\]
RSS(w)

Misst wie gut das Modell zu den Daten passt. Kleineres RSS = besserer Fit.

λ (Lambda)

Hyperparameter, der den Kompromiss zwischen Fit und Koeffizientengröße steuert.

\(\|\mathbf{w}\|_2^2\)

Summe der quadrierten Gewichte. Bestraft große Koeffizienten, hält sie aber nie exakt bei 0.

Wahl von λ — Bias-Variance Tradeoff
Großes λ

Hoher Bias, niedrige Varianz
Koeffizienten werden stark gestaucht. Modell ist einfacher, kann wichtige Muster verpassen.

Kleines λ

Niedriger Bias, hohe Varianz
Wenig Regularisierung. Modell kann overfitting zeigen, nähert sich normaler linearer Regression.

\(\lambda = 0\) entspricht normaler linearer Regression ohne Regularisierung. \(\lambda \to \infty\) drückt alle Koeffizienten gegen 0.
Ridge vs. Lasso
Eigenschaft Ridge (L2) Lasso (L1)
Penalty-Term \(\lambda \cdot \|\mathbf{w}\|_2^2 = \lambda \cdot \sum_i w_i^2\) \(\lambda \cdot \|\mathbf{w}\|_1 = \lambda \cdot \sum_i |w_i|\)
Koeffizienten Klein, aber nie exakt 0 Können exakt 0 werden
Feature Selection Nein — alle Features bleiben Ja — sparse Modelle möglich
Anwendung Alle Features sind relevant Viele irrelevante Features vorhanden

Analytische Lösung
Ridge-Lösung (geschlossene Form)
\[\mathbf{w}^* = (\mathbf{X}^\top\mathbf{X} + \color{#c96a28}{\lambda}\mathbf{I})^{-1}\mathbf{X}^\top\mathbf{y}\]

Verglichen mit der normalen Gleichung \(\mathbf{w} = (\mathbf{X}^\top\mathbf{X})^{-1}\mathbf{X}^\top\mathbf{y}\) wird \(\lambda\mathbf{I}\) zur Matrix \(\mathbf{X}^\top\mathbf{X}\) addiert. Das macht die Matrix immer invertierbar (regularisiert numerisch) und staucht die Gewichte in Richtung 0.
