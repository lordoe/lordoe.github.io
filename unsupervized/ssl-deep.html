// Deep Self-Supervised Learning

// Überblick — Repräsentation lernen

Ein neuronales Netz bildet den Input auf eine niedrig-dimensionale Repräsentation ab: z = fθ(x) ∈ ℝK, mit x ∈ ℝd, K ≪ d. θ sind die Netzparameter (Gewichte & Biases). z heißt auch Embedding, latente Variable oder latente Repräsentation.

Deep Self-Supervised Learning (SSL) lernt diese Repräsentation ohne manuell annotierte Labels — die Supervisionsstruktur kommt aus den Daten selbst (Rekonstruktion, Augmentierungspaare, nächstes Token). Es gibt zwei Hauptfamilien:

Generativ (Rekonstruktion) Der Verlust liegt im Eingaberaum: das Netz rekonstruiert x̂ ≈ x aus der kompakten Repräsentation z.
Auto-encoder VAE Diffusion
\[ \mathcal{L} = \| x - \hat{x} \|^2 \]
↔
Latent-Space Learning Der Verlust liegt direkt im Latenzraum: Paare z, z' werden verglichen oder regularisiert — kein Decoder nötig.
SimCLR CLIP SigLIP VICReg Barlow Twins
\[ \mathcal{L}(z,\, z') \]

// Auto-encoder — nichtlineares PCA

Auto-encoder Rekonstruktion

Encoder fθ und Decoder gφ sind tiefe Netze. Der Encoder komprimiert x auf z ∈ ℝK; der Decoder rekonstruiert daraus x̂. Das Netz wird durch Minimierung des Rekonstruktionsfehlers trainiert — ohne externe Labels.

x ∈ ℝd
→ fθ →
z ∈ ℝK
→ gφ →
x̂ ∈ ℝd
01
Encoder & Decoder
\[ z = f_\theta(x) \in \mathbb{R}^K, \qquad \hat{x} = g_\varphi(z) \in \mathbb{R}^d \]
02
Rekonstruktionsverlust
\[ \mathcal{L}(\theta, \varphi) = \frac{1}{N} \sum_{n=1}^{N} \| x_n - g_\varphi(f_\theta(x_n)) \|^2 \]
03
Analogie zu PCA Ein linearer Auto-encoder mit fθ(x) = WTx und gφ(z) = Wz, trainiert mit MSE, lernt dieselben Unterräume wie PCA. Tiefe (nichtlineare) Aktivierungsfunktionen erzeugen eine nichtlineare Verallgemeinerung von PCA.

// Variational Auto-Encoder (VAE)

VAE Rekonstruktion

Das VAE (Kingma & Welling) erweitert den Auto-encoder um eine probabilistische Latenzstruktur. Statt eines fixen z gibt der Encoder eine Gaußverteilung über z aus.

x
→ qφ(z|x) →
z ~ N(μ, σ²)
→ pθ(x|z) →
x̂

Prior: pθ(z) = N(0, I)

01
Encoder-Ausgabe: Gaußverteilung
\[ q_\varphi(z \mid x) = \mathcal{N}\!\left(z;\; \mu_\varphi(x),\; \sigma^2_\varphi(x)\right) \]
02
Reparameterisierungs-Trick Sampling wird differenzierbar, sodass Gradienten durch μφ und σφ fließen:
\[ z = \mu_\varphi(x) + \sigma_\varphi(x) \odot \varepsilon, \qquad \varepsilon \sim \mathcal{N}(0, I) \]
03
VAE-Verlust: Rekonstruktion + KL-Regularisierung
\[ \mathcal{L} = \| x - \hat{x} \|^2 + \mathrm{KL}\!\left(q_\varphi(z \mid x) \;\|\; \mathcal{N}(0, I)\right) \]
Der KL-Term drängt die Posteriorverteilung zum Standard-Prior — das erzwingt eine geordnete, kompakte Latenzstruktur.

// SigLIP — Binary Cross-Entropy auf Paaren

SigLIP Latent-Space Kontrastiv

Für einen Batch von N Paaren werden beide Seiten eingebettet. Für jedes Paar (i, j) wird ein Ähnlichkeitsscore berechnet und mit Binary Cross-Entropy als Positiv- oder Negativpaar klassifiziert. (Quelle: Zhai et al., SigLIP — Google.)

01
Embeddings & Scores
\[ z_i = f_\theta(x_i), \quad z_i^+ = f_\theta(x_i^+) \]
\[ s^+ = z_i^\top z_i^+ \;(\text{Positivpaar, } i=j), \qquad s^- = z_i^\top z_j^+ \;(\text{Negativpaar, } i \neq j) \]
02
SigLIP-Verlust
\[ \mathcal{L}_\text{SigLIP} = \sum_{s^+} \mathrm{BCE}(1, s^+) + \sum_{s^-} \mathrm{BCE}(0, s^-) \]
\[ \mathrm{BCE}(y, s) = -\bigl[y \log \sigma(s) + (1-y)\log(1-\sigma(s))\bigr], \quad \sigma = \text{Sigmoid} \]

// SimCLR & CLIP — Contrastive Loss (NT-Xent)

SimCLR CLIP Latent-Space Kontrastiv

Zwei Augmentierungen desselben Bildes bilden ein Positivpaar. Der Verlust zieht positive Paare zusammen und schiebt negative (alle anderen im Batch) auseinander. (Quelle: Chen et al., SimCLR.)

01
Kosinus-Ähnlichkeit
\[ s_{ij} = \mathrm{sim}(z_i, z_j^+) = \frac{z_i^\top z_j^+}{\|z_i\|\,\|z_j^+\|} \]
02
Softmax über den Batch
\[ \hat{p}_i = \mathrm{softmax}_j\!\left(\frac{s_{ij}}{\tau}\right) \]
τ ist die Temperatur; sie kontrolliert, wie scharf die Verteilung ist.
03
NT-Xent-Verlust Für jeden Anker i liegt das zugehörige Positiv bei Index i — der Verlust ist die Cross-Entropie:
\[ \ell_i = \mathrm{CE}(i,\, \hat{p}_i) = -\log \hat{p}_{i,i}, \qquad \mathcal{L}_\text{SimCLR} = \frac{1}{N} \sum_{i=1}^{N} \ell_i \]
Alle anderen Einträge zj+ (j ≠ i) dienen als Negative.

// VICReg & Barlow Twins — nicht-kontrastiv

VICReg Barlow Twins Latent-Space Nicht-kontrastiv

Statt expliziter Negativpaare regularisieren diese Methoden die Embedding-Statistik, um den Kollaps (alle z gleich) zu verhindern. Zwei augmentierte Ansichten x, x+ erzeugen Embeddings z, z+ ∈ ℝK.

01
Invarianz-Term — ziehe Positive zusammen
\[ \mathcal{L}_\text{inv} = \frac{1}{N} \sum_{n=1}^{N} \| z_n - z_n^+ \|^2 \]
02
Regularisierung — verhindere Kollaps VICReg: Varianz ≈ 1 auf der Diagonale, Kovarianz → 0 außerhalb:
\[ C_{kk} \approx 1, \qquad C_{kj}^2 \to 0 \quad (k \neq j) \]
Barlow Twins: Kreuzkorrelationsmatrix → Einheitsmatrix:
\[ \mathrm{Corr}(z, z^+) \to I_K \]
03
Gesamtverlust
\[ \mathcal{L} = \mathcal{L}_\text{inv} + \lambda\, \mathcal{L}_\text{reg} \]
Brücke zu PCA: Beide Methoden erzwingen dekorrelierte, einheitsvariante Features — genau die Struktur des PCA-Whitenings.

// Auto-regressiv — Next-Token Prediction

GPT WaveNet Generativ Next-Token

Das Modell lernt, das nächste Token aus der Vergangenheit vorherzusagen: pθ(xt | x<t). Die Labels kommen direkt aus den Daten — das nächste Token — ohne manuelle Annotierung. Grundlage aller GPT-artigen Modelle.

01
Cross-Entropie über die Sequenz
\[ \mathcal{L} = \sum_{t=1}^{T} \mathrm{CE}\!\left(x_t^\text{next},\; \hat{x}_t\right) \]
xtnext ist das wahre nächste Token, x̂t die vorhergesagte Verteilung des Modells bei Position t.
