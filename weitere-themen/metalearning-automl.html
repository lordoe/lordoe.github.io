Motivation — das Algorithmen-Auswahlproblem

Es existieren viele verschiedene ML-Algorithmen — Decision Trees, Naive Bayes, SVM, Neural Networks usw. Jeder Algorithmus hat eigene Parameter. In der Praxis probieren Nutzer typischerweise einige Varianten aus und wählen die beste für ihre Anwendung. Das ist zeitaufwendig und die Systeme sind nicht flexibel für neue Aufgaben. Daraus ergibt sich das Algorithmen-Auswahl/Konfigurationsproblem.

Metalearning adressiert genau dieses Problem: statt Erfahrung über eine einzelne Lernaufgabe zu sammeln, wird Erfahrung über die Performance von ML-Algorithmen auf vielen Anwendungen akkumuliert — dynamisch für Modellauswahl und Methodenkombination.

No Free Lunch (NFL) Theorem
"…for any algorithm, any elevated performance over one class of problems is offset by performance over another class."
— Wolpert & Macready, IEEE Trans. Evolutionary Computation 1(1): 67–82 (1997)
"any two algorithms are equivalent when their performance is averaged across all possible problems."
— Wolpert & Macready, IEEE Trans. Evolutionary Computation 9(6): 721–735 (2005)

Das NFL-Theorem besagt: Es gibt keinen universell besten Algorithmus. Ein Algorithmus, der auf einer Klasse von Problemen besonders gut abschneidet, zahlt dafür mit schlechterer Leistung auf einer anderen Klasse. Wenn man über alle denkbaren Probleme mittelt, sind alle Algorithmen äquivalent.

CCWA
Closed Classification World
Klassifikationsaufgaben in realen Anwendungen bilden eine Teilmenge des Universums. Ein Algorithmus wird gewählt, der auf dieser Teilmenge experimentell besser abschneidet. In der Praxis bevorzugt — Algorithmen werden auf UCI-Repository-Datensätzen verglichen.
OCWA
Open Classification World
Es wird keine Struktur über reale Klassifikationsaufgaben angenommen. Stattdessen wird die Klasse von Aufgaben charakterisiert, auf denen ein neuer Algorithmus gut funktioniert — aufwendiger, aber ehrlicher.
Konsequenz: Da kein Algorithmus universell überlegen ist, brauchen wir Mechanismen zur Algorithmenauswahl — Methoden, die für ein gegebenes Problem den am besten geeigneten Algorithmus identifizieren. Genau das ist die Aufgabe von Metalearning.

Rice's Algorithm Selection Framework (1976)

John R. Rice formalisierte das Algorithmenauswahlproblem mit vier Räumen. Gegeben ein Problem \(x \in P\) mit extrahierten Features \(f(x) \in F\), finde eine Auswahlabbildung \(S(f(x))\) in den Algorithmenraum \(A\), sodass der gewählte Algorithmus \(\alpha \in A\) die Performance \(y(\alpha(x)) \in Y\) maximiert.

Die vier Räume formal
P — Problem Space
x ∈ P
Menge aller Instanzen einer Problemklasse. Z.B. verschiedene Klassifikationsdatensätze aus dem UCI Repository.
F — Feature Space
f(x) ∈ F
Messbare Charakteristika der Probleminstanzen, erzeugt durch eine rechnerische Feature-Extraktion auf P. Meta-Features.
A — Algorithm Space
α ∈ A
Menge aller betrachteten Algorithmen zur Lösung des Problems. Im Metalearning: Menge der Base-Level-Lernalgorithmen.
Y — Performance Space
y(α(x)) ∈ Y
Abbildung jedes Algorithmus auf Performance-Metriken. Z.B. Genauigkeit, Laufzeit, Kompaktheit. Wir optimieren ‖y‖.

Im Metalearning-Kontext gilt: \(A\) ist die Menge von Base-Level-Lernalgorithmen, die Selektionsabbildung \(S\) ist selbst ein Lernalgorithmus, der auf Trainings-Metadaten \(\{\langle f(x), t(x) \rangle : x \in P' \subseteq P\}\) trainiert wird. Dabei ist \(t(x) = \arg\max_{a \in A}\, y(a, x)\) der best-performing Algorithmus auf Instanz \(x\).

Training Metadata: Eine Tabelle mit einer Zeile pro Datensatz — Meta-Features (Anzahl Features, Klassen, Entropie, …) als Eingabe, bester Algorithmus als Ziel. Aus dieser Tabelle lernt \(S\) (z.B. ein Decision Tree), welcher Algorithmus für neue Datensätze am besten geeignet ist.

Feature Space — Meta-Features

Die Wahl geeigneter Meta-Features ist entscheidend: Features müssen Vorhersagekraft bezüglich der Algorithmusperformance besitzen. Es gibt vier Hauptkategorien der Datensatz-Charakterisierung, die in der Praxis eingesetzt werden.

1. Statistische & Informationstheoretische Features

Aus dem Datensatz werden direkt messbare Charakteristika extrahiert. Die Grundannahme: Lernalgorithmen reagieren sensitiv auf die Struktur des Datensatzes. Auch die Trainingsgröße spielt eine wichtige Rolle.

Einfache Metriken
n, p, |C|
Anzahl Instanzen, Anzahl Attribute, Anzahl Klassen, Verhältnis Instanzen/Attribute.
Statistische Metriken
corr, skew, …
Mittlere absolute Korrelation zwischen Features, Korrelation Feature–Ziel, Schiefe der Verteilungen.
Informationstheorie
H(C), H̄
Entropie der Klassenverteilung, mittlere Feature-Entropie, gegenseitige Information Feature–Ziel.
Klassen-Entropie (StatLog-Beispiel):
\[H(C) = -\sum_{c \in C} p(c) \log_2 p(c)\] Hohe Entropie → gleichmäßige Klassenverteilung → schwieriges Problem.
2. Modellbasierte Charakterisierung

Eigenschaften einer Hypothese, die auf dem Problem induziert wurde, werden als indirekte Charakterisierungsform genutzt. Vor allem Decision Trees werden eingesetzt, weil sie interpretierbare Strukturinformationen über den Datensatz liefern.

Nodes per Feature
nodes/p
Wie oft wird jedes Feature im Baum verwendet. Gibt an, welche Features für die Klassifikation wichtig sind.
Max. Baumtiefe
depth_max
Maximale Tiefe des induzierten Entscheidungsbaums. Indikator für Problemkomplexität.
Baum-Imbalanz
imbalance
Wie stark der Baum asymmetrisch ist. Symmetrische Bäume deuten auf ausgewogene Problemstruktur hin.
Idee: Ein Decision Tree wird einmal auf dem Datensatz trainiert — nicht um Vorhersagen zu machen, sondern um dessen Struktur zu beschreiben. Die Baumeigenschaften sind die Meta-Features \(f(x)\).
3. Landmarking

Beim Landmarking wird die Performance einfacher, schneller Lernalgorithmen (Landmarker) als Meta-Feature verwendet. Die Intuition: Jeder Lernalgorithmus hat eine "Expertise-Region" — eine Klasse von Aufgaben, auf der er besonders gut funktioniert. Die Performance eines Landmarkers auf einem Datensatz beschreibt, in welcher Region sich dieser Datensatz befindet.

Landmarker müssen effizient sein — ihr Ausführungsaufwand muss kleiner sein als der Aufwand, alle Algorithmen aus \(A\) zu testen. Sonst könnte man direkt alle Algorithmen evaluieren. Geeignete Landmarker:

  • Naive Bayes
  • OneR (one-rule learner)
  • Lineare Diskriminanzanalyse
  • 1-Nearest Neighbor
  • Decision Stump
Beispiel: Wenn Landmarker i1 und i3 auf einem Datensatz gut abschneiden, i2 aber schlecht, deutet das darauf hin, dass der Datensatz wahrscheinlich in der Expertise-Region von i4 liegt — der weder i1 noch i2 noch i3 ist, aber in der Schnittmenge ihrer Regionen.
Hyperparameter-Optimierung — formale Definition

Die meisten ML-Algorithmen haben Parameter, die vor dem Training festgelegt werden müssen: Anzahl Nachbarn (KNN), Anzahl Schichten (Neural Networks), Anzahl Bäume (Random Forest) usw. Die Konfiguration dieser Parameter ist oft entscheidend für gute Ergebnisse. Der Parameterraum kann diskret, kontinuierlich oder bedingt strukturiert sein — ein großes Suchproblem.

Die formale Definition nach [H2015]: Sei \(A\) ein ML-Algorithmus mit Hyperparametern \(\lambda_1, \ldots, \lambda_n\) und Domänen \(\Lambda_1, \ldots, \Lambda_n\). Der Hyperparameter-Raum ist:

\[\boldsymbol{\Lambda} = \Lambda_1 \times \cdots \times \Lambda_n\]

\(A_{\boldsymbol\lambda}\) bezeichnet Algorithmus \(A\) mit Hyperparameter-Einstellung \(\boldsymbol\lambda\). Die Validierungsverlustfunktion \(\mathcal{L}(A_{\boldsymbol\lambda}, \mathcal{D}_\text{train}, \mathcal{D}_\text{valid})\) misst z.B. die Misklassifikationsrate.

Das Optimierungsproblem — k-fold Cross-Validation

Das Optimierungsproblem unter k-facher Kreuzvalidierung ist das Minimieren der folgenden Blackbox-Funktion:

\[ f(\boldsymbol\lambda) = \frac{1}{k} \sum_{i=1}^{k} \mathcal{L}\!\left(A_{\boldsymbol\lambda},\, \mathcal{D}_\text{train}^{(i)},\, \mathcal{D}_\text{valid}^{(i)}\right) \]
\(\boldsymbol\lambda\)
Hyperparameter-Vektor
Eine Konfiguration aus dem Hyperparameter-Raum \(\boldsymbol\Lambda\). Was wir optimieren.
\(k\)
Anzahl Folds
Typisch: k = 10. Mittelung über k verschiedene Train/Val-Splits für robuste Schätzung.
\(\mathcal{L}\)
Verlustfunktion
z.B. Misklassifikationsrate. Wird auf den Validierungsdaten des i-ten Folds gemessen.
\(f\)
Blackbox-Funktion
Teuer auszuwerten, nicht differenzierbar, ggf. verrauscht. Daher spezielle Suchmethoden nötig.
Herausforderung: Die Funktion \(f(\boldsymbol\lambda)\) ist eine Blackbox — wir können sie auswerten, aber nicht ableiten. Jede Auswertung erfordert das vollständige Training des Modells. Für tiefe Neural Networks kann eine Auswertung Stunden dauern. Ziel: Minimum mit möglichst wenigen Auswertungen finden.
Verbindung: Metalearning + Hyperparameter-Optimierung

Hyperparameter-Optimierung ist ein Spezialfall der Modellauswahl. Die Wahl eines bestimmten Lernalgorithmus kann als Optimierung eines nominalen Hyperparameters betrachtet werden. Vorverarbeitungsschritte (Normalisierung, Encoding) sind ebenfalls nominale Hyperparameter.

Meta-Features können genutzt werden, um die Initialisierung der Hyperparameter für einen neuen Datensatz zu verbessern — statt bei Null zu starten, beginnt man bei einer durch ähnliche Datensätze informierten Konfiguration (Warm-Starting).

Grid Search vs. Random Search — Interaktive Demo

Grid Search testet das kartesische Produkt aller Parameterwerte — exhaustiv und gleichmäßig verteilt. Randomized Search zieht zufällige Konfigurationen. Wenn ein Parameter wenig Einfluss hat (unimportant parameter), verschwendet Grid Search Budget auf dessen Variation, während Random Search mehr Punkte entlang des wichtigen Parameters abdeckt.

Sequential Model-Based Bayesian Optimization (SMBO)

SMBO ist ein allgemeines Framework zur Minimierung von Blackbox-Funktionen \(f\). Ein probabilistisches Surrogatmodell \(\mathcal{M}\) approximiert \(f\) basierend auf bisherigen Auswertungen. Das Modell wird genutzt, um die nächste vielversprechende Konfiguration auszuwählen — ohne \(f\) direkt auszuwerten.

\(\mathcal{M}\) ist ein Regressionsmodell, angepasst auf Datentupeln \((\boldsymbol\lambda, \mathcal{L}(A_{\boldsymbol\lambda}))\). Es sagt eine a-posteriori-Verteilung \(p_\mathcal{M}(f \mid \boldsymbol\lambda)\) vorher.

SMBO Algorithmus — formale Darstellung
Algorithm 1 Sequential Model-Based Bayesian Optimization (SMBO)
1: Initialisiere Modell \(\mathcal{M}\); \(\mathcal{H} \leftarrow \emptyset\)
2: while Zeitbudget für Optimierung nicht erschöpft do
3:    \(\boldsymbol\lambda \leftarrow\) bestimme Kandidatenkonfiguration aus \(\mathcal{M}\)
4:    \(i \leftarrow\) wähle Cross-Validation-Fold
5:    Berechne \(c = \mathcal{L}(A_{\boldsymbol\lambda}, \mathcal{D}_\text{train}^{(i)}, \mathcal{D}_\text{valid}^{(i)})\)
6:    \(\mathcal{H} \leftarrow \mathcal{H} \cup \{(\boldsymbol\lambda, c)\}\)
7:    Aktualisiere \(\mathcal{M}\) gegeben \(\mathcal{H}\)
8: end while
9: return \(\boldsymbol\lambda\) aus \(\mathcal{H}\) mit minimalem \(c\) über alle CV-Folds
          Quelle: [H2015] Hutter et al., Beyond Manual Tuning of Hyperparameters
Acquisition Function — Expected Improvement (EI)

Die Acquisition Function \(a_\mathcal{M}: \boldsymbol\Lambda \to \mathbb{R}\) bestimmt, welche Konfiguration als nächstes ausgewertet wird. Die beliebteste Wahl ist der Expected Improvement (EI) über den bisher besten beobachteten Funktionswert \(f_\text{min}\):

\[ \text{EI}(\boldsymbol\lambda) = \mathbb{E}_{p_\mathcal{M}(f|\boldsymbol\lambda)}\!\left[\max(f_\text{min} - f, 0)\right] \]

EI balanciert Exploitation (Bereiche mit niedrig erwartetem \(f\)) und Exploration (Bereiche mit hoher Unsicherheit). Das probabilistische Modell \(\mathcal{M}\) liefert sowohl den Erwartungswert als auch die Unsicherheit \(p_\mathcal{M}(f \mid \boldsymbol\lambda)\).

Ablauf: Das Surrogatmodell approximiert \(f\) mit einem Unsicherheitsband. Die Acquisition Function wählt den nächsten Punkt mit maximalem EI — oft dort, wo das Modell niedrige Werte vorhersagt oder wo die Unsicherheit hoch ist. Nach jeder Auswertung wird das Modell mit dem neuen Datenpunkt aktualisiert.
SMAC — Sequential Model-Based Algorithm Configuration [H2011]

SMAC ist eine konkrete Implementierung von SMBO, die Random Forests als Surrogatmodell verwendet (anstelle von Gaussian Processes). Das modelliert \(p_\mathcal{M}(f \mid \boldsymbol\lambda)\) über die Variation der Baumvorhersagen im Ensemble.

Anwendungsbereich
Beliebige Algorithmen
Optimierung von Hyperparametern für beliebige Algorithmen, auch harte kombinatorische Problemlöser.
Surrogatmodell
Random Forest
Verwendet Random Forest zur Schätzung von \(p_\mathcal{M}(f \mid \boldsymbol\lambda)\). Robust für gemischte Parameterräume.
Instanzbasiert
Mehrere Datensätze
Basiert auf einer Menge von Instanzen — gut für Algorithmen, die auf verschiedenen Problemen laufen sollen.

Regression Tree fitting — ein Beispiel: Ein Baum wird an Beobachtungen \((\boldsymbol\lambda, \text{runtime})\) angepasst, um die Laufzeit zu schätzen.

Vorteil gegenüber Gaussian Processes: Random Forests können nativ mit kategoriellen und konditionalen Hyperparametern umgehen — wichtig in AutoML, wo Parameter wie "Anzahl Schichten" nur dann relevant ist, wenn als Algorithmus Neural Networks gewählt wurde.

Surrogatmodell gibt Unsicherheit via Baumvarianz:
\[\hat{\mu}(\boldsymbol\lambda) = \frac{1}{B}\sum_{b=1}^B T_b(\boldsymbol\lambda)\] \[\hat{\sigma}^2(\boldsymbol\lambda) = \frac{1}{B-1}\sum_{b=1}^B (T_b(\boldsymbol\lambda) - \hat{\mu})^2\]
AutoML — Automated Machine Learning

AutoML automatisiert den gesamten ML-Anwendungsprozess auf einem Datensatz: Hyperparameter-Optimierung, Algorithmenauswahl und Feature-Selektion/Preprocessing. Das Ziel ist, ML auch für Nicht-Experten zugänglich zu machen und den Aufwand für Experten zu reduzieren.

Auto-Sklearn ([F2017]) — Pipeline-Struktur

Auto-Sklearn (Feurer et al., NIPS 2015) ist ein vollständiges AutoML-System auf Basis von scikit-learn. Es kombiniert SMAC für HP-Optimierung mit Metalearning (Warm-Starting) und Ensemble-Methoden.

Metalearning in Auto-Sklearn: Vor der SMAC-Optimierung werden auf Basis von Meta-Features ähnliche Datensätze aus einer Datenbasis gesucht und deren beste Konfigurationen als Warm-Start-Punkte verwendet. Das reduziert die benötigte Optimierungszeit erheblich.
