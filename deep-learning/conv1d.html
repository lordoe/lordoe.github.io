// Convolutional Sequence Models · Foliensatz 08
01 — Conv1d Definition

Eine eindimensionale Faltung (Conv1d) schiebt einen gelernten Kernel über eine Eingangssequenz und berechnet an jeder Position eine gewichtete Summe. Für Eingabe \(\mathbf{x} \in \mathbb{R}^{T \times D}\), Kernel \(\mathbf{k} \in \mathbb{R}^{K \times D' \times D}\) entsteht Ausgabe \(\mathbf{y} \in \mathbb{R}^{(T-K+1) \times D'}\).

\[ y_t^{(d')} = \sum_{d=1}^{D} \sum_{\tau=0}^{K-1} x_{t+\tau}^{(d)}\, k_\tau^{(d',d)} \]
K — Kernelgröße
D — Eingangskanäle
D' — Ausgangskanäle
K · D · D' — Anzahl Parameter
Interaktiv: Gleitender Kernel — D'=2 Ausgangskanäle
Hier ist D'=2 gewählt (zwei Ausgangskanäle, also zwei Filter k⁽¹⁾, k⁽²⁾ ∈ ℝK×D). Beide Filter gleiten über dieselbe Eingabe und erzeugen je eine eigene Ausgabesequenz. Die Ausgabe ist deshalb kein Vektor, sondern eine Matrix y ∈ ℝ(T−K+1)×D' = ℝ10×2 — eine Spalte (Sequenz) pro Kanal.
Ausgabe y⁽¹⁾₁…y⁽¹⁾₁₀ — Kanal 1 (T−K+1 = 10)
Ausgabe y⁽²⁾₁…y⁽²⁾₁₀ — Kanal 2 (T−K+1 = 10)
Gleitendes Fenster (K=3) — zwei Filter \(k^{(1)}\), \(k^{(2)}\) teilen sich dieses Fenster
Filter \(k^{(1)}\) — erzeugt Kanal \(y^{(1)}\)
Filter \(k^{(2)}\) — erzeugt Kanal \(y^{(2)}\)
Eingabe x ₁…x ₁₂ (T=12, D=1)
Position: t=1
Mehrere Eingangskanäle: Mit \(D=3\) ist jeder Zeitpunkt ein Vektor \(\mathbf{x}_t = (x_t^{(1)}, x_t^{(2)}, x_t^{(3)})\). Bei \(K=3\) sieht jeder Filter deshalb ein Fenster der Form \(K \times D = 3 \times 3\). Für \(D'=2\) gibt es zwei solche Filter, also \(K \cdot D \cdot D' = 3 \cdot 3 \cdot 2 = 18\) Parameter.
Eingabe mit \(D=3\) — dasselbe gleitende Zeitfenster, aber drei Kanäle pro Zeitpunkt
Filter \(k^{(1)} \in \mathbb{R}^{3 \times 3}\)
Filter \(k^{(2)} \in \mathbb{R}^{3 \times 3}\)
Ausgabe \(y^{(1)}_1 \ldots y^{(1)}_{10}\) — Kanal 1, jeder Wert summiert über \(K \cdot D = 9\) Terme
Ausgabe \(y^{(2)}_1 \ldots y^{(2)}_{10}\) — Kanal 2, jeder Wert summiert über \(K \cdot D = 9\) Terme
Aktuelle Ausgabe bei Position t=1
02 — Kernel Interpretationen

Durch geeignete Wahl der Kernelkoeffizienten lässt sich Conv1d als klassische Signalverarbeitung interpretieren. In einem CNN werden diese Kernel aus Daten gelernt.

03 — Padding

Ohne Padding schrumpft die Ausgabe auf \(T - K + 1\). Mit Padding lässt sich die Ausgabelänge erhalten oder kausale Verarbeitung erzwingen.

Ausgabe (10 Zellen, T−K+1=10)
Eingabe (T=12) + Padding-Zellen (grau-gelb)
Keine Nullen hinzugefügt. Ausgabelänge = T − K + 1 = 10.
04 — Rezeptives Feld

Das rezeptive Feld beschreibt, wie viele Eingangs-Zeitschritte eine einzelne Ausgabeposition bei gestapelten Conv1d-Schichten "sehen" kann.

\[ \text{Rezeptives Feld} = 1 + L \cdot (K - 1) \]
Schichten L2
Kernelgröße K3
Rezeptives Feld = 1 + 2 · (3 − 1) = 5
05 — Stride und Dilation

Stride überspringt Eingabepositionen und reduziert die Ausgabelänge. Dilation vergrößert den effektiven Kernelbereich durch Lücken zwischen den Kernelelementen, ohne mehr Parameter zu verwenden.

Ausgabe (10 Zellen)
Eingabe x₁…x₁₂
Standard: Kernel gleitet 1 Schritt, Ausgabe = T − K + 1 = 10.
Dilation d=2: Kernelpositionen k₁, k₂, k₃ greifen auf xt, xt+2, xt+4 zu — effektiver Kernelbereich = d·(K−1)+1 = 2·2+1 = 5, gleiche Parameterzahl K=3.
06 — TCN-Backbone

Ein Temporal Convolutional Network (TCN) stapelt L kausale Conv1d-Schichten, wobei sich die Dilation pro Schicht verdoppelt: Schicht \(\ell\) hat Dilation \(2^\ell\). Dadurch wächst das rezeptive Feld exponentiell mit der Tiefe.

Dilation verdoppelt sich pro Schicht, sodass das rezeptive Feld exponentiell wächst: bei L Schichten mit Dilation 2⁰, 2¹, …, 2^(L-1) ergibt sich RF ≈ 2^L.

PyTorch: nn.Conv1d erwartet Shape (B, D, T), nicht (B, T, D). Transponieren mit x.transpose(1, 2) vor dem Eingang in den TCN-Block.
