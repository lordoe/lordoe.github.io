01 — Conv2d Definition

Eine zweidimensionale Faltung (Conv2d) schiebt einen gelernten Kernel (Filter) über ein 2D-Eingangsbild und berechnet an jeder Position das Skalarprodukt aus Kernel und überlapptem Bildausschnitt. Für Eingabe \(\mathbf{X} \in \mathbb{R}^{H \times W \times C}\) und Kernel \(\mathbf{K} \in \mathbb{R}^{k_h \times k_w \times C}\) entsteht eine Feature Map \(\mathbf{Y} \in \mathbb{R}^{(H-k_h+1) \times (W-k_w+1)}\).

\[ Y_{i,j} = \sum_{c=1}^{C} \sum_{u=0}^{k_h-1} \sum_{v=0}^{k_w-1} X_{\,i+u,\; j+v,\; c}\; K_{u,v,c} \]
k_h × k_w — Kernelgröße
C — Eingangskanäle
S — Stride
P — Padding
Interaktiv: Gleitender Kernel (6×6 → 3×3 Kernel → 4×4)

03 — Padding, Stride & Ausgabegröße

Ohne Padding schrumpft das Bild bei jeder Faltung. Mit Padding P (Nullrand) bleibt die Größe erhalten; Stride S springt mehrere Pixel und reduziert die Auflösung.

\[ O = \left\lfloor \frac{W - k + 2P}{S} \right\rfloor + 1 \]

04 — Pooling (Downsampling)

Pooling fasst räumliche 2×2-Nachbarschaften zusammen und halbiert so die Auflösung. Das reduziert Rechenaufwand und macht das Netz robuster gegen kleine Verschiebungen (Translationsinvarianz). Pooling hat keine lernbaren Parameter.

05 — Kanäle, Feature Maps & Parameter

Eine Conv2d-Schicht lernt \(C_{out}\) Filter, die alle Eingangskanäle gemeinsam abtasten. Jeder Filter erzeugt eine Feature Map. Entscheidend: die Parameterzahl hängt nicht von der Bildgröße ab (Weight Sharing) — anders als bei einer voll verbundenen Schicht.

\[ \#\text{Parameter} = (k_h \cdot k_w \cdot C_{in} + 1)\; C_{out} \]

Weight Sharing: derselbe Filter wird über alle Bildpositionen wiederverwendet. Eine voll verbundene Schicht auf ein 224×224×3-Bild bräuchte Millionen Gewichte pro Neuron — ein 3×3-Conv-Filter über 3 Kanäle nur 3·3·3+1 = 28.
