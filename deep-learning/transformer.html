// Self-Attention & Transformer Architecture

01 — Self-Attention: Intuition

Die Kernidee: der Output yt eines Tokens wird als gewichtete Summe der Value-Vektoren vt' aller anderen Tokens berechnet. Die Gewichte kommen aus der Ähnlichkeit zwischen Tokens.

\[ \mathbf{y}_t = \sum_{t'} A_{t,t'}\, \mathbf{v}_{t'} \quad \text{mit} \quad \mathbf{v}_t = \mathrm{Linear}(\mathbf{x}_t) \]

Die Attention-Matrix A kodiert, welche Tokens auf welche anderen "achten". Ein hoher Wert At,t' bedeutet: Token t zieht viel Information aus Token t'. Da A durch Softmax normiert wird, summiert jede Zeile auf 1.

\[ \sum_{t'} A_{t,t'} = 1 \quad \text{(Softmax-Normierung pro Zeile)} \]

02 — Attention-Matrix: Definition & Visualisierung

\[ A = \mathrm{softmax}_{t'}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\tau}\right), \quad \mathbf{Q} = \mathrm{Linear}(\mathbf{X}),\quad \mathbf{K} = \mathrm{Linear}(\mathbf{X}) \]

Dimensionen der beteiligten Tensoren:

X: (B, T, D) — Batch, Zeit, Features
Q, K: (B, T, d) — projiziert auf Dim. d
A: (B, T, T) — Attention-Gewichte
τ: Temperatur (Skalierung)

03 — Multi-Head Attention

06 — Positional Encoding

Transformers haben keine inhärente Reihenfolge: die Self-Attention ist permutationsinvariant — ohne Positionsinfo behandelt das Modell die Sequenz als ungeordnete Menge (Bag of Tokens).

\[ \mathbf{x}_t \;\leftarrow\; \mathbf{x}_t + \mathbf{p}(t) \]

Der Positionsvektor \(\mathbf{p}(t)\) wird zum Token-Embedding addiert und kodiert die Position t im Kontext. Zwei Varianten:

SINUSOIDAL (FIXED)
\[ p(t)_{2i} = \sin\!\left(\tfrac{t}{10000^{2i/D}}\right) \]

Deterministisch, keine gelernten Parameter, generalisiert auf ungesehene Positionen.

LEARNED (TRAINABLE)
\[ \mathbf{p}(t) = \mathbf{E}_{\mathrm{pos}}[t] \]

Einbettungsmatrix \(\mathbf{E}_{\mathrm{pos}} \in \mathbb{R}^{T_{\max} \times D}\) wird trainiert (z.B. GPT-2).

08 — Cross-Attention (Encoder–Decoder)
